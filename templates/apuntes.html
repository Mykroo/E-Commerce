Apuntes de android

Licencia

Algunas partes del sistema android son desarrolladas en código abierto, así que el código fuente siempre está disponible, sin embargo otras partes son desarrolladas primero en un árbol privado y luego es liberado cuando la siguiente versión de la plataforma está lista

La mayor parte del software android a adoptado la licencia apache 2.0, ésta licencia concede algunos derechos y obligaciones

  • Reproduccion:

    se otorga al licenciatario el derecho de ejecutar y reproducir libremente el software por cualquier medio y cualquier forma

  • Modificación:

    El licenciatario tiene el derecho de modificar libremente el programa, incluido el derecho a traducirlo, así como la posibilidad de crear cualquier tipo de desarrollo e integración derivada del mismo

  • Distribucion:

    El licenciatario tiene el derecho de distribuir libremente el software a tra´ves de cualquier modalidad soporte o formato

  • Comunicación pública:

    El licenciatario podra poner el software a disposición de una pluradidad de personas sin previa distribución de ejemplares físicos a cada una de ellas incluida la ejecución pública del programa de ordenador

  • Sublicenciamiento (en código fuente o código objeto):

    El licenciatario tiene el derecho de sublicenciar libremente el software en código fuente o código objeto

Obligaciones

  • Siempre incluir una copia de la licencia

  • Siempre indicar cualquier modificación

  • Mantener avisos de titularidad

Elementos de un programa android

  1. app/src/main/java

    Carpeta que contiene el codigo fuente de la aplicación clases auxiliares, etc. Los ficheros llava se almacenan en un espacio de nombres

  2. app/build/generated

    : Carpeta que contiene el código generado de forma automática por el sdk

  3. app/src/main/res:

    Directorio para los recursos de la aplicación como los archivos drawable, layout, valores de caden, etc.

  4. app/src/main/res/drawable:

    Directorio para los objetos drawables (como las imagenes) que son diseñadas para pantallas de cierta densidad

  5. app/src/main/res/drawable:

    Directorio par los archivos que definen la interfaz de usuario de la aplicación

  6. app/src/main/AndroidManifest:

    Este ficheero describe la aplicación android en el se indican las actividades intenciones, servicios y provedores de contenido de la aplicación. También se declaran los permisos que requerira la aplicación y se indica la versión mínima de android para poder ejecutarla.

  7. build.gradle:

    Contiene información necesaria para la compilación del proyecto, por ejemplo la versión del sdk de android utilizada para compilar, la minima versión de android que soportara la aplicación, referencias a las librerías externas, etcetera
